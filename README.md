# Today I Learn 페이지 만들기

### HTML

1. 나만의 갤러리에서 사진 9장은 HTML 태그 중 어떤 요소로 표현하는 것이 좋을까? 이유는 뭐라고 생각했는가?
    - <img> 태그로 표현하는 것이 좋다. 사진 9장은 이미지에 해당하므로 images 폴더 안에 넣고 상대경로를 기입하여 표기해줘야한다고 생각했기 때문이다.
2. 뼈대 코드를 기반으로 이미지를 로드하는 방식은 크게 2가지로 나뉘는데 무엇이고 어떻게 사용하고 있는가?
    - img src로 직접 삽입하는 방식과, css  background-image로 스타일로 삽입하는 방법이 있다.
3. 시맨틱 태그(예: **`<header>`**, **`<main>`**, **`<section>`**, **`<footer>`** 등)로 문서를 구분할 때, 어떤 기준으로 섹션을 나누고 태그를 적용하는 것이 좋을까?
    - 시맨틱 태그의 목적은 의미를 명확하게 하여 가독성을 향상 시키는 데에 있다. 로고나 제목에 해당하는 부분은 header, 화면의 하단부는 footer로 정의한 후 그 사이의 스페이스들을 어떻게 배치할 지 생각하고 글, 콘텐츠 등의 속성에 맞게 태그를 적용하는것이 좋다.
4. **`<img>`** 태그의 **`alt`** 속성은 왜 필요하며, 어떻게 작성하는 것이 좋을까?
    - 시각 장애인이 스크린 리더를 사용할 때 이미지 정보를 텍스트로 이해하기 위해 필요하다. 또한, 에러가 나서 이미지가 표시되지 않을 때 표시될 텍스트의 역할도 하기 때문에 필요하다. 이런 목적성을 고려했을 때, 사진의 의미에 맞는 간결한 설명으로 작성하는 것이 좋다.
5. 이미지를 로드할 때, 상대 경로(relative path)와 절대 경로(absolute path)를 혼용하면 어떤 문제가 발생할 수 있을까?
    - 개발 환경과 배포 환경(서버)는 다르기 때문에 경로가 일치하지 않아 오류가 발생하기 쉽다. 또한, 이곳 저곳에 상대 경로, 절대 경로가 혼용되어 있다면 변경이 생길 때 유지보수하기 어렵다는 문제가 발생할 수 있다.
6. **`<meta>`** 태그의 viewport 설정과 Open Graph 태그는 각각 어떤 역할을 하는가?
    - viewport 설정은 모바일 기기에서도 정상 작동하도록 반응형 설정의 역할을 하고, Open Graph는 링크 공유 시에 표시할 제목, 이미지를 설정하는 역할을 한다.

### CSS

1. 나만의 갤러리에서 9장의 이미지를 보기 좋게 배치하려면, Flexbox와 CSS Grid 중 어떤 것이 더 적합할까? 그 이유는?
    - 9장의 이미지를 격자 형태로 정갈하게 배치하려면 2차원 레이아웃에 최적화된 CSS Grid가 훨씬 적합합니다.
    Grid는 행과 열을 동시에 제어하여 고정된 비율의 격자를 만들기 쉽고, 이미지 사이의 간격(gap)을 직관적으로 관리할 수 있기 때문입니다.
2. 태그 선택자, 클래스 선택자, ID 선택자는 각각 언제 사용하는 것이 적절할까?
    - 태그는 페이지 내 모든 해당 요소(예: p, li)에 공통 스타일을 입힐 때, 클래스는 여러 요소에 동일한 디자인을 재사용할 때 사용합니다.
    ID는 페이지 내에서 단 하나만 존재하는 고유한 요소를 특정할 때 주로 사용합니다.
3. CSS를 HTML 내에 직접 작성하는 것과 별도 파일로 분리하는 것의 차이는 무엇일까?
    - HTML 내 직접 작성은 별도 파일 로드 없이 빠르게 적용되지만, 코드가 복잡해지면 유지보수가 어렵고 재사용성이 떨어집니다.
    별도 파일 분리는 여러 HTML 페이지에서 하나의 CSS를 공유할 수 있어 관리가 효율적이며, 브라우저 캐싱을 통해 로딩 속도를 개선할 수 있습니다.

### JavaScript

1. **`document.getElementById()`**와 **`document.querySelector()`**의 차이는 무엇인가?
    - **`getElementById()`**는 인자로 유일한 요소의 Id를 받아서 찾습니다.
    - **`querySelector()`**는 괄호 속 제공한 선택자의 첫 번째 요소를 반환합니다.

    - 버튼 클릭시 다크모드로 전환하기
3. **`innerHTML`**로 DOM을 조작할 때 주의해야 할 점은 무엇일까
    - 사용자 입력을 그대로 innerHTML에 넣으면 악성 스크립트가 실행될 수 있으므로, 단순 텍스트는 textContent를 쓰는 것이 안전합니다.
